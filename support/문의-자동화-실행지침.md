## 매 실행 시 절차
1. **새 문의 수집**: milbigyo 받은편지함에서 아직 처리되지 않은 메일 검색.
   - 예: `in:inbox is:unread -label:문의/검토대기` (자동/뉴스레터성 발신자 제외).
2. 각 메일에 대해 `get_thread` 로 본문을 읽는다.
3. **분류**: 「문의-답변-지식베이스.md」의 카테고리 표 기준으로 1개 카테고리 지정.
4. **답변 초안 작성**:
   - 지식베이스의 사실만 근거로, 해당 카테고리 템플릿을 메일 내용에 맞게 다듬는다.
   - 확신 불가 시 "확인 후 회신드리겠습니다"로 남긴다.
   - `create_draft` 로 **해당 스레드에 대한 회신 초안**을 milbigyo 계정에 저장.
   - 스팸 카테고리는 초안 작성 생략.
5. **라벨 부착**: 카테고리 라벨 + `문의/검토대기` 를 메일/스레드에 부착(`label_thread`).
6. **요약 보고**: 이번 실행에서 처리한 문의 목록(보낸사람·제목·카테고리·초안작성여부)을 한 줄씩 정리해 보고.
